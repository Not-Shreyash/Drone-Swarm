Python-Based Drone Control

1. Introduction to MAVSDK

MAVSDK is a high-level software development kit that allows developers to communicate with drones using programming languages such as Python, C++, Swift and Java.

MAVSDK uses the MAVLink communication protocol to exchange commands and telemetry data with the flight controller.

With MAVSDK, we can:

  • Connect to a drone.
  • Read GPS coordinates.
  • Monitor battery and health.
  • Arm and disarm the drone.
  • Take off and land.
  • Create autonomous missions.
  • Control multiple drones.

2. Why Python for Drone Programming?

Python is one of the most popular languages for robotics and autonomous systems. It is simple to learn and allows developers to quickly test algorithms without writing large amounts of code.

Language Difficulty Speed Common Use
Python Easy Medium Rapid development and testing
C++ Hard Very Fast Flight controllers and ROS nodes

5. Creating a Virtual Environment

Ubuntu 24.04 prevents users from installing Python packages directly into the system Python installation. Therefore, we create a virtual environment.

Step 1: Create the virtual environment

python3 -m venv venv

Step 2: Activate the environment

source venv/bin/activate

After activation, the terminal changes from:

  • Before:
user_laptop:~/DronePython$
  • After:
(venv) user_laptop:~/DronePython$

Common Errors and Solutions

Error 1: Command not found

  • Cause: Python is not installed.
  • Solution:
sudo apt install python3 python3-pip python3-venv

Error 2: "externally-managed-environment"

  • Ubuntu 24.04 blocks package installation into the system Python.
  • This was the exact error encountered during this project.
  • Solution: Use a virtual environment. 
error: externally-managed-environment

Notes

  • Always activate the virtual environment before running drone scripts.
  • The virtual environment must be activated every time a new terminal window is opened.
  • Installing MAVSDK directly into the system Python is not recommended.
  • The virtual environment does not affect Ubuntu or ROS packages.

6. Installing MAVSDK

MAVSDK (MAV Software Development Kit) is a library that allows Python programs to communicate with PX4 using the MAVLink protocol. Without MAVSDK, we would have to manually create and send MAVLink packets, which is difficult and error-prone.

Step 1: Activate the virtual environment

cd ~/DronePython
source venv/bin/activate

After activation, your terminal should look similar to: 

(venv) user@user_laptop:~/DronePython$

Step 2: Install MAVSDK

The installation process may take a few minutes depending on your internet connection.

7. Writing Our First Program: connect.py

Now that MAVSDK has been installed successfully, we are ready to write our first Python program. The objective of this program is simple:

  • Create a drone object.
  • Connect to PX4.
  • Wait until the connection is established.
  • Print a success message.

Step 1: Create the Python file

touch connect.py

You can open the file using VS Code: 

code connect.py

Step 2: Write the following code

import asyncio
from mavsdk import System


async def main():

    print("Creating drone object...")

    drone = System()

    print("Connecting to PX4...")

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for drone connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("✅ Connected to the drone!")

            break


if __name__ == "__main__":

    asyncio.run(main())

Step 3: Understanding the Code

Line Meaning
import asyncio Imports Python's asynchronous library.
from mavsdk import System Imports the MAVSDK drone interface.
drone = System() Creates a drone object.
await drone.connect(...) Connects to PX4 through MAVLink.
async for ... Continuously listens for updates.
state.is_connected Checks whether the drone is connected.
break Stops the loop after connecting.

Step 4: Run the program

Before running the script, make sure:

  • PX4 SITL is running. (seperate terminal)
  • Gazebo is open.(seperate terminal)
  • The virtual environment is activated.
python connect.py

Expected Output

Creating drone object...

Connecting to PX4...

Waiting for drone connection...

✅ Connected to the drone!

⚠️ Common Errors and Solutions

Error 1: Connection timeout

  • Cause: PX4 SITL or Gazebo is not running.
  • Solution:

    • In a new terminal

cd ~/PX4-Autopilot

make px4_sitl gz_x500

Error 2: "Connection using udp:// is deprecated"

  • This warning was encountered during this project.
  • Replace: 
udp://0.0.0.0:14540

with:

udpin://0.0.0.0:14540

📝 Notes

  • The drone does not need to be flying to establish a connection.
  • PX4 continuously sends telemetry data even while the drone is on the ground.
  • The program uses asynchronous programming, which allows it to wait for updates without freezing.
  • Port 14540 is used by PX4 SITL to communicate with MAVSDK.

8. Understanding Telemetry

Telemetry is the data continuously sent by the drone to the ground station or to our Python program. Even when the drone is not flying, PX4 keeps transmitting information such as GPS location and health status.

Examples of telemetry data:

  • Latitude
  • Longitude
  • Absolute altitude
  • Relative altitude
  • Battery percentage
  • Flight mode
  • Health status

Step 1: Create telemetry.py

touch telemetry.py
code telemetry.py

Step 2: Write the following code

import asyncio

from mavsdk import System


async def main():

    drone = System()

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("Connected!")

            break

    print("Reading telemetry data...\n")

    async for position in drone.telemetry.position():

        print(position)


if __name__ == "__main__":

    asyncio.run(main())

Step 3: Run the program

python telemetry.py

Expected Output

Connected!

Reading telemetry data...

latitude_deg=47.397742

longitude_deg=8.545594

absolute_altitude_m=488.02

relative_altitude_m=0.00

Why does telemetry work even when the drone is on the ground?

During this project, a common question was:

  • "How is telemetry data available if the drone is not flying?"

The answer is simple. The drone does not need to fly in order to send telemetry data. As soon as PX4 starts, its sensors begin transmitting information continuously.

9. Health Monitoring

Before taking off, PX4 checks whether all important systems are working properly. This process is called a health check or preflight check. If any check fails, the drone will refuse to arm.

Before arming, PX4 verifies:

  • GPS availability.
  • Home position initialization.
  • Sensor health.
  • Communication status.
  • Battery status.

Step 1: Create health.py

touch health.py
code health.py

Step 2: Write the following code

import asyncio

from mavsdk import System


async def main():

    drone = System()

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("Connected!")

            break

    print("Checking drone health...\n")

    async for health in drone.telemetry.health():

        print(health)

        break


if __name__ == "__main__":

    asyncio.run(main())

Step 3: Run the program

python health.py

Expected Output

Connected!

Checking drone health...

is_global_position_ok=True

is_home_position_ok=True

Understanding the Health Parameters

Parameter Meaning
is_global_position_ok Checks whether the GPS position is valid.
is_home_position_ok Checks whether PX4 has initialized the home position.

Why are health checks important?

Imagine the drone tries to take off before GPS initialization is complete. The autopilot will not know:

  • Its current position.
  • Its home location.
  • Where to return during an emergency.

Therefore, PX4 blocks arming until these checks pass.

⚠️ Common Errors and Solutions

Error 1: Drone refuses to arm. 

WARN [commander] Arming denied:
Resolve system health failures first
  • Cause: One or more health checks failed.
  • Solution: Wait until all health parameters become True.

Error 2: No connection to the ground station. 

WARN [health_and_arming_checks]
Preflight Fail: No connection to the GCS
  • GCS stands for Ground Control Station.
  • In our setup, QGroundControl acts as the Ground Control Station.
  • Start QGroundControl before attempting to arm the drone.

📝 Notes

  • Health monitoring should always be performed before takeoff.
  • A successful connection does not guarantee that the drone is ready to fly.
  • PX4 may refuse to arm if GPS or home position initialization is incomplete.
  • QGroundControl serves as the Ground Control Station (GCS).
