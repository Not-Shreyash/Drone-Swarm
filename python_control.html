Python-Based Drone Control

1. Introduction to MAVSDK

MAVSDK is a high-level software development kit that allows developers to communicate with drones using programming languages such as Python, C++, Swift and Java.

MAVSDK uses the MAVLink communication protocol to exchange commands and telemetry data with the flight controller.

With MAVSDK, we can:

  • Connect to a drone.
  • Read GPS coordinates.
  • Monitor battery and health.
  • Arm and disarm the drone.
  • Take off and land.
  • Create autonomous missions.
  • Control multiple drones.

2. Why Python for Drone Programming?

Python is one of the most popular languages for robotics and autonomous systems. It is simple to learn and allows developers to quickly test algorithms without writing large amounts of code.

Language Difficulty Speed Common Use
Python Easy Medium Rapid development and testing
C++ Hard Very Fast Flight controllers and ROS nodes

7. Writing Our First Program: connect.py

Now that MAVSDK has been installed successfully, we are ready to write our first Python program. The objective of this program is simple:

  • Create a drone object.
  • Connect to PX4.
  • Wait until the connection is established.
  • Print a success message.

Step 1: Create the Python file

touch connect.py

You can open the file using VS Code: 

code connect.py

Step 2: Write the following code

import asyncio
from mavsdk import System


async def main():

    print("Creating drone object...")

    drone = System()

    print("Connecting to PX4...")

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for drone connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("✅ Connected to the drone!")

            break


if __name__ == "__main__":

    asyncio.run(main())

Step 3: Understanding the Code

await drone.connect(...) Connects to PX4 through MAVLink.
async for ... Continuously listens for updates.
state.is_connected Checks whether the drone is connected.
break Stops the loop after connecting.

9. Health Monitoring

Before taking off, PX4 checks whether all important systems are working properly. This process is called a health check or preflight check. If any check fails, the drone will refuse to arm.

Before arming, PX4 verifies:

  • GPS availability.
  • Home position initialization.
  • Sensor health.
  • Communication status.
  • Battery status.

Understanding the Health Parameters

Parameter Meaning
is_global_position_ok Checks whether the GPS position is valid.
is_home_position_ok Checks whether PX4 has initialized the home position.

Why are health checks important?

Imagine the drone tries to take off before GPS initialization is complete. The autopilot will not know:

  • Its current position.
  • Its home location.
  • Where to return during an emergency.

Therefore, PX4 blocks arming until these checks pass.

⚠️ Common Errors and Solutions

Error 1: Drone refuses to arm. 

WARN [commander] Arming denied:
Resolve system health failures first
  • Cause: One or more health checks failed.
  • Solution: Wait until all health parameters become True.

Error 2: No connection to the ground station. 

WARN [health_and_arming_checks]
Preflight Fail: No connection to the GCS
  • GCS stands for Ground Control Station.
  • In our setup, QGroundControl acts as the Ground Control Station.
  • Start QGroundControl before attempting to arm the drone.

📝 Notes

  • Health monitoring should always be performed before takeoff.
  • A successful connection does not guarantee that the drone is ready to fly.
  • PX4 may refuse to arm if GPS or home position initialization is incomplete.
  • QGroundControl serves as the Ground Control Station (GCS).

Step 1: Create mission1.py

touch mission1.py

Step 2: Write the following code

import asyncio

from mavsdk import System


async def main():

    drone = System()

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("Connected!")

            break

    print("Waiting for health checks...")

    async for health in drone.telemetry.health():

        if health.is_global_position_ok and health.is_home_position_ok:

            print("Drone is ready.")

            break

    print("Arming...")

    await drone.action.arm()

    print("Taking off...")

    await drone.action.takeoff()

    print("Hovering for 10 seconds...")

    await asyncio.sleep(10)

    print("Landing...")

    await drone.action.land()


if __name__ == "__main__":

    asyncio.run(main())

Step 3: Run the mission

python mission1.py

Understanding the Program

Command Purpose
drone.connect() Connects Python to PX4.
drone.telemetry.health() Checks whether the drone is ready.
drone.action.arm() Arms the drone.
drone.action.takeoff() Starts the takeoff sequence.
asyncio.sleep(10) Keeps the drone hovering.
drone.action.land() Commands the drone to land.

Commands vs Drone State

A command being accepted does not mean that the drone has already completed the action.

For example: 

await drone.action.takeoff()

This line only means that PX4 has accepted the takeoff command. It does not guarantee that the drone has already reached its target altitude.

Question Encountered During Development

A common beginner question is:

  • "If the program executes takeoff(), has the drone already finished taking off?"

The answer is no. The drone may still be on the ground or may just be beginning its climb.

Error 2: Arming denied.

Resolve system health failures first
  • Cause: One or more health checks failed.
  • Solution: Wait for GPS and home position initialization.

📝 Notes

  • Autonomous missions are simply sequences of commands.
  • The drone must pass health checks before arming.
  • The takeoff command does not wait for the drone to reach its target altitude.

11. Open-Loop vs Closed-Loop Control

In the previous chapter, our drone followed a fixed sequence of commands:

Arm
⬇
Take off
⬇
Wait 10 seconds
⬇
Land

The program never checked whether the drone had actually reached the desired altitude. It simply trusted that everything would work correctly. This type of control is called open-loop control.

Open-Loop Control

In open-loop control, the controller sends commands but does not verify whether the drone has reached the desired state.

Characteristic Open-Loop Control
Uses sensor feedback No
Checks the actual position No
Simple to implement Yes
Reliable Less reliable

Closed-Loop Control

In closed-loop control, the drone continuously sends sensor data back to the program. The program uses this feedback to decide what to do next.

Send command
⬇
Drone moves
⬇
Sensors measure position
⬇
Python reads telemetry
⬇
Decision

Example

Suppose we want the drone to climb to an altitude of 5 meters.

Open-loop approach: 

await drone.action.takeoff()

await asyncio.sleep(10)

We simply assume that the drone reaches the target altitude.

Closed-loop approach: 

while current_altitude < 5:

    read telemetry

    wait

Now the program waits until the drone actually reaches the desired altitude.

Why do real drones use closed-loop control?

  • Wind may push the drone.
  • GPS measurements may fluctuate.
  • Motors may behave differently.
  • Battery voltage changes over time.
  • Sensor readings are never perfect.

Without feedback, autonomous flight would be unsafe.

Connection to Drone Swarms

Closed-loop control is the foundation of:

  • Waypoint navigation.
  • Circular trajectories.
  • Formation flying.
  • Obstacle avoidance.
  • Multi-drone coordination.

In the upcoming chapters, we will use telemetry and feedback to make the drone follow a circular trajectory.

🎯 Challenge for the Reader

📝 Notes

  • Open-loop control sends commands without checking the result.
  • Closed-loop control uses sensor feedback.
  • Most autonomous robots use closed-loop control.

12. Our First Closed-Loop Program

In the previous mission, we assumed that the drone had completed takeoff after waiting for a fixed amount of time. However, in robotics, assumptions are dangerous. In this chapter, we will use telemetry data to continuously monitor the drone's altitude and wait until it actually reaches the desired height.

Step 1: Create altitude_monitor.py

touch altitude_monitor.py
code altitude_monitor.py

Step 2: Write the following code

import asyncio

from mavsdk import System


TARGET_ALTITUDE = 2.0


async def main():

    drone = System()

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("Connected!")

            break

    print("Waiting for health checks...")

    async for health in drone.telemetry.health():

        if health.is_global_position_ok and health.is_home_position_ok:

            print("Drone is ready.")

            break

    print("Arming...")

    await drone.action.arm()

    print("Taking off...")

    await drone.action.takeoff()

    print("Waiting to reach target altitude...")

    async for position in drone.telemetry.position():

        altitude = position.relative_altitude_m

        print(
            f"Current altitude: "
            f"{altitude:.2f} m"
        )

        if altitude >= TARGET_ALTITUDE:

            print(
                "Target altitude reached!"
            )

            break

    print("Landing...")

    await drone.action.land()


if __name__ == "__main__":

    asyncio.run(main())

Understanding TARGET_ALTITUDE

TARGET_ALTITUDE = 2.0

This variable stores the altitude that we want the drone to reach. The value is measured in meters.

Reading altitude data

altitude = position.relative_altitude_m

The drone continuously sends its current altitude. The value is measured relative to the takeoff point.

Making decisions

if altitude >= TARGET_ALTITUDE:

    print("Target altitude reached!")

    break

This is the heart of closed-loop control. Instead of blindly waiting, the program continuously measures the drone's state and makes decisions based on sensor data.

How will we make the drone fly in a circle?

A circle can be approximated by many small waypoints. The Python script will repeatedly calculate: 

x = R × cos(θ)

y = R × sin(θ)

where:

Symbol Meaning
R Radius of the circle
θ Current angle
x X coordinate
y Y coordinate

Example

Suppose: 

R = 5 m

θ = 90°

Then: 

x = 5 × cos(90°) = 0

y = 5 × sin(90°) = 5

Therefore, the waypoint is: 

(0, 5)

Step 2: Generate the points

import math

radius = 5

for angle in range(0, 361, 30):

    theta = math.radians(angle)

    x = radius * math.cos(theta)

    y = radius * math.sin(theta)

    print(
        f"Angle: {angle:3d}°"
        f"  ->  "
        f"x = {x:.2f}, "
        f"y = {y:.2f}"
    )

Understanding the for loop

for angle in range(0, 361, 30):

This means:

  • Start from 0°.
  • Stop at 360°.
  • Increase by 30° every iteration.

Therefore, the loop will generate: 

0°, 30°, 60°, 90°, 120° ...

... 300°, 330°, 360°

Why do we need many points?

A circle cannot be represented by only four points.

  • More points produce a smoother circle.
  • Fewer points produce a polygon.
  • The drone follows one waypoint after another.

16. Sending Position Setpoints to the Drone

Until now, our drone only moved vertically. To follow trajectories such as squares, circles, and swarm formations, we must tell the drone where to go. Such target locations are called setpoints.

What is a setpoint?

A setpoint is simply a desired position. For example: 

x = 5

y = 3

z = -2

This means:

  • Move 5 meters forward.
  • Move 3 meters to the right.
  • Fly 2 meters above the ground.

⚠️ Why is altitude negative?

PX4 internally uses the NED coordinate system:

  • X → North
  • Y → East
  • Z → Down

Because the Z-axis points downward, altitude values are negative. 

z = -2

means:

Fly 2 meters upward.

The command used to send a setpoint

await drone.offboard.set_position_ned(
    PositionNedYaw(
        5.0,
        0.0,
        -2.0,
        0.0
    )
)

Understanding PositionNedYaw

Parameter Meaning
5.0 X coordinate
0.0 Y coordinate
-2.0 Altitude
0.0 Yaw angle

  • Circular trajectories are implemented by continuously updating these setpoints.

Step 2: The circular trajectory program

import asyncio
import math

from mavsdk import System

from mavsdk.offboard import (
    PositionNedYaw,
    OffboardError
)


RADIUS = 60.0

ALTITUDE = -2.0

STEP = 2


async def main():

    drone = System()

    await drone.connect(
        system_address="udpin://0.0.0.0:14540"
    )

    print("Waiting for connection...")

    async for state in drone.core.connection_state():

        if state.is_connected:

            print("Connected!")

            break

    print("Waiting for health checks...")

    async for health in drone.telemetry.health():

        if (
            health.is_global_position_ok
            and
            health.is_home_position_ok
        ):

            print("Drone is ready.")

            break

    print("Arming...")

    await drone.action.arm()

    print("Sending initial setpoint...")

    await drone.offboard.set_position_ned(

        PositionNedYaw(
            0.0,
            0.0,
            ALTITUDE,
            0.0
        )

    )

    print("Starting Offboard mode...")

    try:

        await drone.offboard.start()

    except OffboardError as error:

        print(
            f"Offboard failed: {error}"
        )

        return

    print("Flying in a circle...")

    for angle in range(
        0,
        361,
        STEP
    ):

        theta = math.radians(
            angle
        )

        x = RADIUS * math.cos(
            theta
        )

        y = RADIUS * math.sin(
            theta
        )

        print(
            f"x={x:.2f}, "
            f"y={y:.2f}"
        )

        await drone.offboard.set_position_ned(

            PositionNedYaw(

                x,

                y,

                ALTITUDE,

                angle

            )

        )

        await asyncio.sleep(
            0.5
        )

    print("Stopping Offboard mode...")

    await drone.offboard.stop()

    print("Landing...")

    await drone.action.land()


if __name__ == "__main__":

    asyncio.run(main())

Understanding the parameters

Variable Meaning
RADIUS Radius of the circle
ALTITUDE Flight height
STEP Angular increment

What does STEP control?

STEP = 2
  • Smaller STEP → smoother circle.
  • Larger STEP → polygon-like motion.
  • More points → more computations.

⚠️ Common Errors and Solutions

Error: 

Offboard failed
  • PX4 simulation is not running.
  • The drone was not armed.
  • The MAVSDK connection failed.

Error: 

The drone moves strangely.
  • The coordinate system may be misunderstood.
  • The radius may be too large.
  • The sleep time may be too small.

Error: 

ImportError:
No module named mavsdk.offboard
  • Check whether MAVSDK is installed in the virtual environment.
  • Activate the venv before running the program.

📝 Notes

  • A circle is simply many waypoints.
  • The drone follows each waypoint one by one.
  • The trajectory is generated using trigonometry.
  • This is the foundation of swarm flight and formation control.

19. Easy Trajectory Visualization Using Python

Gazebo does not automatically display the trajectory followed by the drone. A simple beginner-friendly solution is to save the drone coordinates in a text file and later plot them using Python. This method does not require ROS 2 or any Gazebo plugins.

Step 1: Create a file to store the trajectory

Open circle_flight.py and add the following code before the circular trajectory loop. 

trajectory_file = open(

    "trajectory.txt",

    "w"

)

This command creates a new file named trajectory.txt.

  • If the file does not exist, Python creates it.
  • If the file already exists, its contents are erased.
  • The "w" mode stands for write mode.

Step 2: Save the drone coordinates

Locate the following section inside your trajectory loop: 

x = RADIUS * math.cos(theta)

y = RADIUS * math.sin(theta)

Immediately below it, add: 

trajectory_file.write(

    f"{x} {y}\n"

)

During every iteration of the loop, Python writes the current coordinates into the file.

Example:

  • First point → 3.0 0.0
  • Second point → 2.95 0.52
  • Third point → 2.82 1.03

Updated trajectory loop

for angle in range(

    0,

    theta = math.radians(

        angle

Step 3: Close the file

Before the landing command, add: 

trajectory_file.close()

Closing the file ensures that all coordinates are saved correctly.
